# 1分ツール 運営基準

最終更新: 2026-07-17

## 運営の目的

検索や直接訪問で「文章に困った」「1分だけ暇をつぶしたい」と感じた人が、迷わず使えて再訪できるサイトへ育て、十分な利用が確認できた段階で収益化する。

## Codexが担当すること

- Search ConsoleとCloudflare Web Analyticsの確認
- サイト戦略と改善対象の選定
- 文章ツール、例文、1分コンテンツ、ゲームの企画・実装
- 既存コンテンツの品質改善と重複整理
- スマホ表示、回遊、アクセシビリティ、速度、不具合の改善
- ビルド、リンク、主要操作、PC・スマホ表示の検証
- GitHubへのcommit・push、Cloudflare Pagesの本番反映確認
- 運営結果、問題、ユーザーに必要な作業の報告

## 日次運営

1. 作業前にGitの状態と本番稼働を確認する。
2. 取得できる範囲で検索とアクセスの数値を確認する。
3. データとサイト全体の状態から、その日に価値が高い改善を選ぶ。
4. 件数を固定せず、関連する変更をひとまとまりで実施する。
5. 全検証に成功した変更だけを本番へ反映する。
6. 数値、変更、検証、コミット、ユーザー作業の有無を報告する。

## 週次運営

- 表示回数、クリック、アクセス、閲覧ページの7日推移を確認する。
- 伸びたページ、表示だけあるページ、使われていないページを分ける。
- 関連ページの品質改善、タイトルや導線、ツール機能をまとめて見直す。
- 新規追加より、検索意図と利用場面が明確な既存ページを優先する。
- サイト全体のリンク、サイトマップ、旧ドメイン転送を確認する。

## 月次運営

- サイトのテーマとコンテンツ構成が利用データに合っているか評価する。
- 低品質、重複、保守できない機能を整理する。
- 収益化の準備状況を評価し、広告、アフィリエイト、テンプレート販売などから適した方法を提案する。
- 有料契約や本人情報が必要な段階で、ユーザーへ判断材料と操作手順を提示する。

## 自動で行わないこと

- ドメイン、Cloudflare、Google、GitHubなどの所有権や認証情報の変更
- 有料サービスの契約、広告出稿、料金の発生
- 税務・法務上の最終判断
- 個人情報、本人確認情報、振込先情報の入力
- 検証を通らない変更の公開
- force push、履歴破壊、未完了のユーザー変更の上書き

## 収益化へ進む条件

固定のアクセス数だけで決めず、次を総合して判断する。

- 検索表示とクリックが継続して増えている
- 特定のツールやページに繰り返し利用がある
- 人気分野と訪問者の目的が説明できる
- 広告や販売導線を追加しても使いやすさを壊さない
- プライバシー、問い合わせ、運営情報が整っている

条件が整うまでは、利用価値と検索流入の基盤を優先する。
