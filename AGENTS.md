# 1分ツール Codex運用ガイド

## プロジェクト概要

- 日本語の無料ツール・例文・1分コンテンツ・ミニゲームを提供する静的サイト。
- 本番URLは `https://1mintool.com/`。
- GitHubは `maakuu2011-tech/one-minute-tools`。
- `main` へのpushでCloudflare Pagesが自動デプロイする。
- 旧URL `https://one-minute-tools.pages.dev/` は本番URLへ301転送される。

## ユーザーとの進め方

- ユーザーへの説明は日本語で、専門用語を避けて短く伝える。
- コード、検証、GitHubへの反映、本番確認はCodexが担当する。
- ログイン、支払い、本人確認など、本人しかできない操作だけをユーザーへ依頼する。
- ユーザーに操作を依頼するときは「どこを開き、何を押し、何を入力するか」を順番に示す。
- 変更は十分にまとまってからcommit・pushし、細かいデプロイを連続させない。

## コンテンツ方針

- 数だけ増やさず、実際にコピペして使える自然な日本語を優先する。
- 文章例は状況、相手、次の行動が分かる具体的な内容にする。
- 例文の送信前に、氏名、日付、事実関係を確認できる導線を用意する。
- SEOだけの変更は更新履歴へ掲載しない。
- マスコットは案内や状態表示に使い、画面の主目的を邪魔する装飾にはしない。
- スマホで最初に目的の機能へ到達できることを優先する。
- 新規ページを追加するときは、既存ページとの重複と内部リンクを確認する。

## 開発・検証

WindowsではPowerShellの実行ポリシーを避けるため、`npm.cmd` を使う。

```powershell
npm.cmd install
npm.cmd run dev
npm.cmd run build
```

公開前に最低限、次を確認する。

1. `npm.cmd run build` が成功し、全静的ページが生成される。
2. `git diff --check` で問題がない。
3. 変更ページをPC幅とスマホ幅で確認する。
4. 内部リンク、ページ内リンク、コピーなどの主要操作を確認する。
5. push後に本番URLが200を返し、新しい内容が表示されることを確認する。
6. `sitemap-index.xml` と旧ドメインの301転送を壊していないことを確認する。
